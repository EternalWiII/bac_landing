Про проєкт

Розробка інформаційної системи для зберігання та керування файлами в хмарному середовищі. Бекенд реалізовано на мові Java із використанням сучасних фреймворків та технологій. Проєкт дозволяє забезпечити безпечний та зручний доступ до даних через API.

Актуальність теми

У сучасному цифровому світі зростає потреба у надійному та доступному з будь-якого місця зберіганні даних. Хмарні сховища стають стандартом для бізнесу, освіти та особистого використання. Розробка власного рішення дозволяє краще контролювати дані, забезпечувати безпеку та гнучкість у роботі з файлами.

Мета та завдання

Мета: Розробити інформаційну систему хмарного сховища з веб-інтерфейсом та REST API, яка дозволяє зберігати, переглядати, завантажувати та видаляти файли користувачів.

  • Проаналізувати аналоги та технологічні вимоги до хмарного сховища
  • Розробити архітектуру системи (бекенд, API, базу даних, безпеку)
  • Реалізувати REST API для взаємодії з клієнтською частиною
  • Застосувати механізми автентифікації та авторизації користувачів
  • Протестувати систему на працездатність та навантаження

Методологія дослідження

У розробці використовувалися Java 17, Spring Boot для реалізації логіки сервера; Spring Security для безпечної автентифікації користувачів; PostgreSQL для зберігання метаданих файлів та користувачів; Архітектура системи побудована за принципами REST.

Очікувані результати

  • Повнофункціональний REST API для роботи з файлами
  • Безпечна авторизація та автентифікація (JWT)
  • Підтримка ролей користувачів (адміністратор / звичайний користувач)
  • Масштабована архітектура для подальшого розширення
  • Демонстраційний клієнт або документація для підключення фронтенду
